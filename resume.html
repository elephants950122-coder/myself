🎯 雙向痛點解決與 UX 設計
  • 真實場域提案：為瑪利MAMA手作麵包量身打造數位輔具，取代傳統 LINE 群組人工接單的高錯率與高成本。
  • 顧客端 (時段預約鎖定)：系統依據產能設定時段上限，額滿自動反灰。實現強制分流，減少顧客等待時間。
  • 員工端 (認知友善設計)：針對庇護工場員工特性，改採「時段區塊化看板」與「數據圖塊化」，降低心智障礙員工多工處理的焦慮感。
💻 技術實作與系統架構
  • 原生 Android 開發：使用 Kotlin 撰寫，並落實 MVVM 架構，將介面(View)與資料(Model)分離，提升協作效率。
  • 防超賣機制：串接 Firebase Firestore 確保多人同時下單時不會發生超賣，實現精準產能控管。
  • 自動化通知：克服邏輯與 API 串接瓶頸，成功實作下單與出餐完成的 Email 自動通知機制。
💡 業界驗證與個人反思

從對 Kotlin 語法感到陌生，到能熟練運用 Android Studio 與 GitHub 協作，整個過程帶來了極大成就感。 我們的「產能導向」結合「認知友善」系統設計，獲得了現場評審——授課教授 楊子青老師與瑪利亞基金會雲端發展室 林浩棟主任的青睞並榮獲第二名。 這次專案讓我深刻體會，工程師不僅要會寫代碼，更要具備同理心，將技術轉化為解決真實痛點的數位輔具。
